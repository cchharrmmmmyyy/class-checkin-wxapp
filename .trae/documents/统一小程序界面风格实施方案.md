# 统一小程序界面风格实施方案

## 1. 核心设计风格分析
基于登录界面的设计，确定统一风格的核心元素：
- **主题色**：墨绿色系（#2E8B57 为主色）
- **背景**：渐变绿色背景 + 半透明装饰元素
- **卡片样式**：磨砂玻璃效果（backdrop-filter: blur）
- **按钮设计**：绿色渐变 + 圆角 + 阴影效果
- **字体**：统一的字体大小和颜色方案
- **布局**：简洁、居中的布局结构

## 2. 实施步骤

### 2.1 优化全局样式（app.wxss）
- 提取登录界面的核心样式作为全局样式
- 定义统一的颜色变量
- 创建通用的容器、卡片、按钮样式
- 定义全局动画效果

### 2.2 统一页面布局结构
- 为所有页面创建一致的布局框架
- 包括导航栏、内容区域、底部区域
- 确保页面间距和边距统一

### 2.3 迁移样式到各页面

#### 学生页面组
- student.wxml/student.wxss：修改打卡和请假按钮样式
- student-detail.wxml/student-detail.wxss：优化详情页卡片样式
- leave-apply.wxml/leave-apply.wxss：统一表单样式
- leave-records.wxml/leave-records.wxss：优化记录列表样式

#### 教师页面组
- teacher.wxml/teacher.wxss：统一教师面板样式

### 2.4 创建通用样式类
- `.global-container`：统一的页面容器
- `.glass-card`：磨砂玻璃卡片效果
- `.primary-btn`：主按钮样式
- `.secondary-btn`：次要按钮样式
- `.input-field`：统一输入框样式
- `.section-title`：统一标题样式

## 3. 预期效果
- 所有页面保持一致的墨绿色主题
- 统一的卡片和按钮设计
- 简洁、现代的视觉效果
- 减少代码冗余，便于维护
- 提升用户体验的一致性

## 4. 实施顺序
1. 优化全局样式文件 app.wxss
2. 迁移样式到学生主页面
3. 迁移样式到学生详情页
4. 迁移样式到请假相关页面
5. 迁移样式到教师页面
6. 测试所有页面的一致性和响应性

通过以上实施步骤，将实现小程序所有页面风格的统一，保持简洁美观的设计，同时提高代码的可维护性。
