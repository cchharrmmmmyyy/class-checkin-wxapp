# 极简Web管理员功能实现计划

## 1. 需求分析
- 实现最简管理员功能
- Web页面登录，无需小程序
- 核心功能：用户管理（查看、添加、修改、删除）
- 尽量减少代码量和复杂度

## 2. 技术方案
- 基于现有Flask + SQLite架构
- 单文件HTML实现管理界面
- 极简CSS样式
- 基本JavaScript交互
- 无需复杂的模板引擎

## 3. 实现步骤

### 3.1 数据库修改
- 在`database.py`中添加1个管理员示例账户

### 3.2 创建管理员模块
- 创建`admin.py`，实现管理员蓝图
- 添加核心API：
  - Web登录接口
  - 获取用户列表
  - 添加/修改/删除用户

### 3.3 添加单个Web页面
- 创建`admin.html`，整合登录和管理功能
- 极简CSS样式
- 基本JavaScript交互

### 3.4 集成到主应用
- 在`app.py`中注册管理员蓝图

## 4. 极简功能设计

### 4.1 管理员账户
- 用户名：admin
- 密码：admin123
- 角色：admin

### 4.2 核心功能

| 功能 | 描述 |
|------|------|
| 管理员登录 | 简单的用户名密码登录 |
| 用户列表 | 表格展示所有用户信息 |
| 添加用户 | 表单添加新用户 |
| 修改用户 | 编辑现有用户信息 |
| 删除用户 | 删除指定用户 |

### 4.3 API设计

| 路径 | 方法 | 功能 |
|------|------|------|
| `/admin` | GET | 管理员页面 |
| `/api/admin/login` | POST | 登录验证 |
| `/api/admin/users` | GET | 获取用户列表 |
| `/api/admin/users` | POST | 添加/修改用户 |
| `/api/admin/users/<user_id>` | DELETE | 删除用户 |

## 5. 实现效果
- 单HTML页面实现所有功能
- 管理员登录后显示用户列表
- 支持添加、修改、删除用户
- 极简设计，无多余功能

## 6. 代码修改点
- `backend/database.py` - 添加管理员账户
- `backend/app.py` - 注册管理员蓝图
- `backend/admin.py` - 实现核心功能
- `backend/admin.html` - 单页面Web界面

- 开发速度快
- 无需复杂的前端框架
- 功能完整，满足基本需求
