## 文档使用建议

1. **新开发者**：建议先阅读`开发指南.md`，了解项目整体架构和开发环境搭建，然后再参考`API文档.md`进行具体开发。

2. **二次开发**：先阅读`开发指南.md`中的二次开发部分，了解项目的扩展机制，然后根据需要查阅`API文档.md`。

3. **前端开发**：主要参考`API文档.md`，了解后端提供的接口和数据格式。

4. **后端开发**：需要同时参考`开发指南.md`和`API文档.md`，了解项目结构和接口设计。
